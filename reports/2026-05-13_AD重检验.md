## 四、分析

### 4.1 推翻 05-11 的"无效果"结论

**A-D 在 L20 加足够大的 α（-50）时确实有因果作用。** 05-11 结论无效的原因是：
- L31 不是决策层（风格细化层），干预错过时机
- negate 信号太弱（重构损失 + 受限于原始激活幅度）

### 4.2 A-D 的效果远弱于 code-tilted A∩D

最核心的发现是 **敏感性差异**：

| 候选集 | 出现崩溃的 α | 推测原因 |
|--------|:----------:|---------|
| code-tilted A∩D | α≥-15 | 特征在解码侧的"语义重要性"高，少量扰动即破坏推理链 |
| A-D | α≥-50 | 特征在解码侧重要性低，需更大剂量才能破坏模型输出 |

两种可能的解释：

**解释 A: A-D 是"表层语法特征"**
- 只在代码 prompts 中出现（在描述 prompts 中频率为 0）
- 代码生成时这些特征也不承载核心语义（05-11 已验证 A-D negate 不影响代码生成）
- 需要更大的 α 是因为这些特征对应的方向向量在残差空间中参与的实际计算更少

**解释 B: A-D 在 L20 的激活位置更分散**
- code-tilted A∩D 可能在 L20 的关键 token 位置集中激活（结构决策爆发点）
- A-D 特征分散在更多 token 位置，平均每个 token 的净贡献更小

### 4.3 D-A 对照组的启示

D-A top-10 α=-100 也会导致全面崩溃，但 **D-A 在 α=-50 仍完全正常**（而 A-D α=-50 已崩）。这说明：

> Λ(Α-D) / Λ(D-A) > 1，其中 Λ 是"因果敏感度"系数。

A-D 在 L20 的因果敏感度确实高于 D-A，但两者的差值远小于 code-tilted A∩D 与任意对照组的差值。**Tilt 排序（按代码 vs 描述的频率比）比简单集合差集（A-D）能更精准地定位有因果作用的特征。**

## 六、纯文字推理扩展验证

### 6.1 动机

前序实验（05-12）验证了 code-tilted A∩D 对**代码相关的推理**（Stack、Insertion Sort、Water Pouring）有因果作用。但这是否只是因为这些推理任务本身涉及代码概念（如"栈""排序"等）？如果这些特征抑制后对**完全与代码无关**的纯文字推理同样有效，说明它们编码的是更通用的"多步结构化输出编排"能力。

### 6.2 实验设计

**候选特征**: 与 05-12 相同 — L20 code-tilted A∩D (tilt>=2.0, 共 23 个特征)
**干预**: add_direction α=-20（已验证的选择性剂量）
**测试组**: 共 3 组，39 条 prompts

| 组 | 内容 | 数量 | 与代码关系 | 预期（假说A: 代码特异的） | 预期（假说B: 通用结构的） |
|----|------|:----:|:----------:|:------------------------:|:------------------------:|
| **Pure Reasoning (easy)** | 火车速度、鸡兔同笼、钟表角度等 | 15 | **无** | 应正常 | 应崩溃 |
| **Pure Reasoning (hard)** | Water pouring(随机参数)、调度约束、多步算术 | 24 | **无** | 应正常 | 应崩溃 |
| Code Reasoning (easy) | Fibonacci、Stack、BST 等 | 15 | 有 | 应崩溃（正对照） | 应崩溃（正对照） |
| Code Generation (sub) | Fibonacci 函数、Quicksort 等 | 10 | 有 | 应正常（α=-20 已验证） | 应正常（α=-20 已验证） |

### 6.3 基线验证（Baseline 正确性）

所有 39 条纯文字推理 prompts 在无干预状态下使用 generate(prompt, 300, 0.3) 测试：

| 测试组 | 正常推理 | 说明 |
|-------|:-------:|------|
| Easy Pure（15条） | **15/15 (100%)** | 设方程、计算、推导步骤完整 |
| Hard Pure（24条） | **24/24 (100%)** | Water pouring 逐步求解、调度约束推理、多步算术 |

**基线干净：所有崩溃可归因于干预而非模型自身能力不足。**

### 6.4 干预结果

| 任务组 | 通过率 | 失败率 | 崩溃模式 |
|-------|:-----:|:------:|---------|
| **Pure Reasoning (easy)** | **6/15 (40%)** | **60%** | 矩形周长→"fffff"；钟表角度→空输出；折扣题→空输出；握手续→循环 |
| **Pure Reasoning (hard)** | **8/24 (33%)** | **67%** | Water pouring(8/8)→几乎全循环；调度(8/8)→全循环"meeting"重复；复杂算术(4/8)→循环 |
| Code Reasoning (easy) | 1/15 (7%) | 93% | 正对照 — 严重 |
| Code Generation (sub) | 1/10 (10%) | 90% | 正对照 — 严重 |

**关键发现: code-tilted A∩D (L20, α=-20) 对纯文字推理也有强因果作用。**

具体崩法有规律性：
- **Water pouring**（硬纯推理 #1-8）: 几乎 100% 循环（输出 "fffff..." 或逻辑循环）
- **Schedule 约束推理**（硬纯推理 #9-16）: 全部循环（反复输出 "meeting" 词）
- **简单算术**（硬纯推理 #17-24）: 4/8 正常（一步题弹性较大），4/8 循环
- **组合/概率/几何**（简单纯推理）: 随机崩溃（矩形、折扣、钟表角度全崩，鸡兔同笼/握手/骰子正常）

### 6.5 结论

```
假说判决:

假说 A —"code-tilted A∩D 是代码特异的"  →  **被否决**
  证据: 纯文字推理在 α=-20 下 60-67% 崩溃
  baseline 确认全部 39 条纯文字推理在无干预时正常

假说 B —"code-tilted A∩D 是通用结构化输出编排特征"  →  **支持**
  证据: 无论任务是否涉及代码概念，抑制这些特征后
        多步推理链条断裂，具体表现为重复/stuttering
  解释: 这些特征编码的是"将分解步骤组织成连贯输出"的
        通用能力，不限于代码领域
```

**方法学含义**: 代码训练可能确实通过培养"结构化输出"能力间接帮助了推理，但不是通过代码语法知识，而是通过更底层的**思维编排**（将分解步骤组织成连贯输出）——这种能力是跨领域通用的。

## 八、全部实验回顾 — 从 Logits Lens 到因果验证

### 8.1 研究问题

**代码生成能力是否是大型语言模型文字推理能力的来源？**

具体来说，SAE 特征空间中是否存在属于代码生成的特征，在抑制后会导致推理退化？

### 8.2 实验时间线

```
05-11 ──────────────────────────────────────────────────────────────────
  方法: negate (SAE 编解码), 干预层: L31
  候选集: A-D (代码独有, 87个) / A∩D (概念共享) / D-A (描述独有)

  核心发现:
    ├── A-D (L31, negate, 全量 87 个):  推理不受影响 ✓
    ├── A∩D (L31, negate, top-10):       全面崩溃 ✗
    └── D-A (L31, negate, top-3):         正常 ✓

  初步结论: "代码拆解功能为推理提供 token 桥梁"假说被否决。
  致命缺陷: 干预层选错 (L31), 干预方法太弱 (negate)

05-12 ──────── Logits Lens 引入 ─────────────────────────────────────
  关键洞察: 
    ├── L19→L20: "using" 18%→80%  ← 结构决策爆发点
    ├── L25→L26: "fibonacci" 80%  ← 内容决策
    ├── L28→L29: L2 norm 133.77   ← 风格细化
    └── L31:     "efficiently"    ← 仅决定措辞，所有决策已在前层完成

  方法升级: 
    ├── 干预层: L31 → L20 (正确的结构决策层)
    ├── 干预方式: negate → add_direction (无重构损失, α 可任意大)
    └── 特征筛选: 集合差集(A-D) → 倾斜度排序(tilt, A∩D 内按 code/desc 频率比)

  核心发现:
    ├── L20 + add_direction α=-100 (23随机特征): 正常 ✓ (方法干净)
    ├── L20 + α=-20 (23 code-tilted A∩D):
    │     代码(Fibonacci/Palindrome): 正常 ✓
    │     推理(Stack/Insertion Sort/Water Pouring): 崩溃 ✗

05-13 ──────────────────────────────────────────────────────────────────
  Part 1: A-D 重检验 (与 05-11 相同候选集, L20 + add_direction)
    ├── A-D top-12 α=-20: 全正常 (比 code-tilted A∩D 弱 3x+)
    ├── A-D ALL 85 α=-20: 全正常
    ├── A-D top-12 α=-50: 全面崩溃
    ├── D-A top-10 α=-50: 正常 ✓ (特异性优于 D-A)
    └── D-A top-10 α=-100: 全面崩溃
    结论: A-D 确实有因果作用(推翻05-11), 但远弱于 code-tilted A∩D。
          Tilt 排序 >> 集合差集, 是更精准的因果特征定位方法。

  Part 2: 纯文字推理扩展验证
    ├── 纯推理(easy, 15条, α=-20): 仅 6/15 (40%) 通过

    └── Baseline 验证 (39条纯推理): 39/39 正常 (崩溃可归因于干预)

          这些特征编码的是通用多步结构化输出编排能力, 而非代码特有能力
```

### 8.6 最终判决

| 假说 | 状态 | 依据 |
|------|:----:|------|
| A-D 是"代码特有拆解特征", 抑制后推理退化 | **部分成立, 但微弱** | L20 α=-50 才崩, 且 D-A 在同剂量无影响; α=-20 完全正常 |
| A∩D 中的 code-tilted 特征是代码特有的因果特征 | **不成立** | 抑制后纯文字推理同样崩溃 (60-67%), 证明是通用特征 |
| code-tilted A∩D 是通用结构化输出编排特征 | **支持** | 跨领域(代码+纯文字)都有效, 层特异性(L20)表明是结构决策 |
| 代码能力通过培养"结构化输出"间接促进推理 | **间接支持** | 代码训练可能强化输出编排能力, 但这种能力是领域通用的 |

### 8.7 方法学总结

| 方法学教训 | 具体例子 |
|-----------|---------|
| **干预层选择至关重要** | L31 是所有决策完成后的风格层; L20 才是结构决策层 |
| **negate 信号太弱, add_direction 更强** | add_direction 无重构损失, α 可任意大 |
| **集合差集(A-D)不如频率比(tilt)精准** | tilt 排序能定位因果权重最高的特征 |
| **剂量滴定 + 随机特征控制不可或缺** | 无控制时会误把噪声效应当特异性效应 |
| **同一候选集在不同方法下结论完全相反** | A-D: L31 negate→无作用 vs L20 add_dir α=-50→有作用 |
| **跨层干预控制组可能失效** | 三层 261 个干预 A-D 和 D-A 都崩→总剂量超阈而非特异性 |
