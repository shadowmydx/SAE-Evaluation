## 实验版本说明

本报告包含两个版本的实验：

| 版本 | 比较基准 | 问题 |
|---|---|---|
| **V1 (旧版)** | C ∩ (A-B) vs C ∩ (B-A) | B 与 C 完全不共享领域，代码特征占优是必然结果 |
| **V2 (修正版)** | C ∩ (A-D) vs C ∩ (A∩D) | D 是同概念纯描述，正确控制概念共享 |

**V1 的结论存在严重混淆，V2 是可信版本。**

---

## 假说

代码生成任务和文字推理任务共享底层的"过程拆解"机制。模型生成代码时，需要将问题分解为步骤序列（循环、递归、条件分支），这些步骤在 SAE 特征空间中形成可观测的"拆解特征"。

**核心假说**: 当模型执行文字推理时，同样会激活这些代码特有的拆解特征 — 即模型通过"回忆代码中的拆解模式"来完成推理。

---

## 实验设计

### Prompt 分组

| 组 | 内容 | 数量 | 角色 |
|---|---|---|---|
| **A (Code)** | 代码生成 | 30 | 提供"代码拆解特征" |
| **D (Description)** | 同概念纯事实描述 | 15 | **关键控制组**: 与 A 共享概念但无代码逻辑 |
| **C (Reasoning-code)** | 代码相关推理（Fibonacci、stack 等） | 15 | 实验组: 代码概念 + 推理 |
| **C' (Reasoning-pure)** | 纯推理（行程、年龄、概率等） | 15 | 对照组: 仅推理，无代码概念 |

### 候选集设计

核心创新：用 D 控制概念共享。

```
A-D = 代码独有特征（"拆解过程特征"）← 假说关心的
A∩D = 概念共享特征（代码和描述都涉及的概念）← 语义干扰
```

C 在这两个集上的重叠对比，决定假说是否成立。

### 关键指标

- **代码拆解特征 (A-D)**: 排除概念共享后，真正的代码独有特征
- **概念共享特征 (A∩D)**: 代码和纯描述共有的语义特征
- **拆解/概念比**: avg(A-D) / avg(A∩D) × (|A∩D|/|A-D|) — 考虑候选集大小后的标准化比例
- **D 控制组**: D prompt 自身在 A-D 上的重叠 → 应接近 0，否则控制失效

### 总体

| 层 | A 候选 | B 候选 | 平衡 |
|---|---|---|---|
| 15 | 1,341 | 1,365 | 1.02x |

| 层 | C 代码比 | C 知识比 | 主导比 | C' 代码比 | C' 知识比 | 主导比 |
|---|---|---|---|---|---|---|
| 15 | 1.09% | 0.84% | **1.29x** | 0.85% | 0.91% | **0.93x** |
| 24 | 1.37% | 0.84% | **1.64x** | 0.87% | 0.77% | **1.13x** |
| 31 | 1.20% | 0.58% | **2.12x** | 0.85% | 0.64% | **1.33x** |

**V1 结论**: C 中代码特征占优 1.29x~2.12x，C' 中也有优势 0.93x~1.33x，弱支持假说。

**V1 漏洞**: B 是"化学符号""首都"等与 C 完全无关的知识。C 讲 Fibonacci、stack、BST，跟 B 半点关系没有，代码特征多于知识特征几乎是必然的。

## 干预实验 — 因果验证

**运行命令**: 
```bash
# Step 1: 差异发现 (code vs description)
python3 surgenry/discover_features.py \
  --prompt-a "Write a function to compute the nth Fibonacci number" \
  --prompt-b "The Fibonacci sequence is a series where each number is the sum of the two preceding ones" \
  -l 31 --output discovered_code_vs_desc.json

# Step 2: 频率排序
python3 surgenry/rank_features.py \
  --input discovered_code_vs_desc.json \
  --name code_vs_desc --group-a "code-specific" --group-b "description-specific" \
  --prompts-a surgenry/data/prompts/code.json \
  --prompts-b surgenry/data/prompts/descriptions.json \
  --num-prompts 30 --topk 10

# Step 3: 干预
python3 surgenry/steer_generation.py steer-negate \
  -i ranked_code_vs_desc.json -l 31 --side a --top 10 \
  --prompt "..." -m 100 -t 0.3
```

### 干预设计

| 组 | 操作 | 预期（若假说成立） |
|---|---|---|
| **代码专用特征 (A-D)** | Negate top-3 / top-10 | 推理应显著退化 |
| **描述专用特征 (D-A, 控制)** | Negate top-3 | 推理不应受影响 |
| **概念共享特征 (A∩D)** | Negate top-10（从 30 个 code + 15 desc prompt 的全量扫描中排序） | 推理可能受影响（概念丢失），但生成基本功能应保留 |
| **基线** | 无干预 | 作为对比基准 |

### 候选特征

#### 代码专用 (A-D) — 来自 Step 1-2 的标准流程

Layer 31 的 seed-prompt 差异发现:
```
A-only (code-specific): 87 features
B-only (description-specific): 87 features
Shared (A∩B):             13 features
```

经跨 30 code prompt 频率排序后，top-3 代码专用特征频率为 **1.0, 0.93, 0.93**（高置信度）。

#### 概念共享 (A∩D) — 来自 rank_shared.py 的全量扫描

```
扫描 30 code + 15 description prompt 后的交集:
  |A| = 1224, |D| = 991, |A∩D| = 315

Top-10 共享特征（按两侧总出现频率排序）:
  #21348  total=45
  #24065  total=44
  #24156  total=39
  #44476  total=35  (也在 code-specific top-10 中!)
  ...
```

**注意**: `#44476` 同时出现在 A-D top-10 和 A∩D top-10 中。这是一个重要的提示：A-D 和 A∩D 的划分不是绝对的，排序方法不同会导致特征"跨界"（seed-prompt 判定 vs 全量扫描判定）。

| 条件 | 输出质量 | 关键内容 |
|---|---|---|
| **Baseline** | 正确 | 正确模拟 Push 1, Push 2, Pop → 2, Push 3, Pop → 3, Pop → 1, 最终栈为 [1] |
| **Negate top-3 code** | **正确** | 同上，完整推理 |
| **Negate top-3 desc (control)** | 正确 | 同上 |
| **Negate top-5 shared** | 正确 | 同上 |

#### Bubble Sort 推理

| 条件 | 输出质量 | 关键内容 |
|---|---|---|
| **Baseline** | 正确 | 正确模拟第一轮: [5,3,8,1,9]→[3,5,8,1,9]→[3,5,1,8,9] |
| **Negate top-3 code** | **正确** | 同上，完整分步推理 |
| **Negate top-10 code** | **正确** | 正确 |
| **Negate ALL 87 code** | **正确** | **全量 negate，推理依然完整** |
| **Negate top-3 desc (control)** | 正确 | 同上 |
| **Negate top-5 shared** | 正确 | 同上 |

#### Fibonacci 推理

| 条件 | 输出质量 | 关键内容 |
|---|---|---|
| **Baseline** | 正确 | 5th=5, 6th=8, 正确推导 |

#### 跨层干预（Layers 15+24+31，261 个特征）

| 条件 | Bubble sort | 结论 |
|---|---|---|
| **Negate A-D ALL 三层** | **崩溃**（仅输出特殊字符） | 不能归因于代码特征 |
| **Negate D-A ALL 三层（控制）** | **同样崩溃** | **261 个干预超过残差流容错阈值，与特征类型无关** |

#### 单层对比：Layer 15 vs Layer 31

| 层 | Negate A-D 全部 | Negate D-A 全部（控制） | 结论 |
|---|---|---|---|
| **Layer 15（早期）** | 崩溃 | **同样崩溃** | Layer 15 对任何特征干扰均脆弱 |
| **Layer 31（深层）** | **正确** | 正确 | Layer 31 容错能力强，代码特征无因果作用 |

跨层干预的关键发现：**261 个干预全部崩溃，但控制组（D-A 同数量）也一样崩溃**。这不是代码特征的特异性效应，而是跨层同时修改太多特征后残差流被破坏。

而单层对比更有力：在 Layer 31（对特征干扰最鲁棒的层），A-D 全量 87 个 negate 不影响推理；Layer 15 虽然 A-D 和 D-A 都崩，但控制组也崩，说明这是层的脆弱性而非代码特征的作用。

### 干预实验完整证据链

| 干预条件 | 特征类型 | 量级 | 推理结果 | 对照组结果 | 解析 |
|---|---|---|---|---|---|
| **L31 top-10** | A-D（代码独有） | 10 | 正确 | D-A top-3: 正确 | 高层代码特征少量 negate 无影响 |
| **L31 ALL** | A-D（代码独有） | 87 | **正确** | — | **全量 negate 推理仍完好。这是最干净的因果证据** |

| **L15 ALL** | A-D（代码独有） | 89 | 崩溃 | D-A ALL 89: 同崩 | 浅层干预：总剂量问题，非特异性 |
| **L15+L24+L31 ALL** | A-D（代码独有） | 261 | 崩溃 | D-A ALL 261: 同崩 | 跨层干预：总剂量超阈值，非特异性 |

| 干预对象 | 代表什么 | 所有 prompt 效果 | 因果含义 |
|---|---|---|---|
| **Negate A-D top-10** | 代码独有 top-10（高频） | 无影响 | 代码特征在推理中无因果作用 |
| **Negate A-D ALL 87** | **代码独有全部** | **无影响** | **全量 negate 推理依然完整，证据不可辩驳** |
| **Negate D-A top-3** | 描述独有特征（控制组） | 无影响 | 对照组预期 |
| **Negate A∩D top-10** | 概念共享特征（全量扫描排序） | **全面崩溃** | **但这些不是"概念特征"，而是通用语言能力特征** |

**关键发现：A∩D 的崩溃不能说明"推理和代码共享概念"。**

为什么？因为如果 A∩D 真的是"代码和描述共享的概念特征"：

1. neagte A∩D 虽然会破坏相关推理（失去概念），但 **"The capital of France is" 这种与代码/描述都没关系的生成不应该受影响**
2. 但事实上连最简单的基础知识 recall 都只能输出一个标点
3. 所以这些 A∩D 特征不是"代码和描述共享的概念"——而是 **"几乎所有连贯文本都需要的通用语言能力特征"**。它们之所以在 code 和 description 中都出现，只是因为它们在几乎所有文本中都出现。

| 特征集 | 真实含义 | 在推理中的角色 | 因果证据 |
|---|---|---|---|
| **A-D（代码独有）** | 确实是与代码生成相关的特定特征 | **无角色** | Negate top-10 → 无影响; ALL 87 → 无影响; 跨层 261 → 但控制组也崩 |
| **D-A（描述独有）** | 特定于事实描述的语气/句式特征 | 无角色 | Negate → 无影响 |
| **A∩D（概念共享）** | **不是"概念共享"**，是通用语言能力特征 | 文本生成的基础组件 | Negate → 全面崩溃 |

**这是本实验最重要的方法学教训：A∩D ≠ "共享的概念特征"，更可能是通用高频特征在各 prompt 集合中的交集。**

---

## 最终结论

### V2 修正后的核心发现

| 发现 | 结论 |
|---|---|
| **D 控制组完美** | 候选集分割方法有效 |
| **概念共享特征在推理中占主导** | 推理激活的特征主要是语义概念特征，而非代码拆解特征 |
| **V1 的主导结论是概念干扰的假象** | 因为 B 与 C 无关，代码特征多于知识是必然的 |
| **C vs C' 差异很小** | 代码推理和纯推理在特征层面差异不大，仅差 2-3 个特征 |
| **Layer 15 拆解/概念最接近** | 浅层可能真有一部分拆解机制重叠（1.10x） |

### 为什么 V1 和 V2 结论相反？

V1 中"代码特征主导"是因为 B（不相关知识集）是**错误的基线**。代码特征 > 知识特征只说明 C 涉及代码领域概念，不说明"推理借用了代码拆解机制"。

V2 引入 D（同概念描述集）后，**概念共享才是主导因素**。代码独有特征在推理中没有显示特殊的优势。

### 对原始假说的评估

**假说 "代码拆解为推理提供了 token 桥梁" — 被实验结果否决。**

| 实验阶段 | 结果 | 对假说的意义 |
|---|---|---|
| **相关性 V1**（A-B vs B-A） | 代码特征占优 1.29x~2.12x | 看似支持，但 B 与 C 领域不匹配，结论无效 |
| **相关性 V2**（A-D vs A∩D） | A∩D 主导 | 但 A∩D 并非概念特征，而是通用语言特征 |
| **因果干预 A-D**（Negate top-10 → ALL 87） | 三种推理任务均完全不受影响 | **代码特征在推理中无因果作用** |
| **因果干预 A∩D**（Negate top-10 A∩D） | **全面崩溃，包括最基础的知识 recall** | A∩D 是通用语言能力特征，非概念特征 |

否定假说的最强证据来自干预实验：从 top-10 到 **全部 87 个**代码专用特征全部 negate 后，Stack LIFO、Bubble sort 第一步、Fibonacci 序列推理全部保持正确，与 baseline 无差异。全量 negate 下模型依然能正确分步推理——这说明代码拆解特征对推理**不具有因果作用**。

此外，为了排除"推理任务太简单，模型在背答案"的质疑，我们生成了 **随机参数化的 Hard Reasoning Prompts**（`surgenry/data/prompts/hard_reasoning_code.json`），确保每个问题不可能通过记忆回答。在 Layer 31 全量 negate 87 个 A-D 特征后，**随机 Stack LIFO、随机 Insertion Sort、随机 Sequence Induction（递推规则+种子随机）、随机 Graph 最短路、随机 Water Pouring 全部正确**。这排除了"假阴性"的可能性。

#### A-D 特征的标签修正：非"代码独有"，部分为"广义过程拆解"

为进一步理解 A-D 特征的语义，我们引入过程拆解控制组 **E（Procedural Description）：11 个非代码但需要分步描述的任务（菜谱、组装家具、滴定实验、换轮胎等）**。扫描 E 在 A-D 上的重叠：

| 层 | A-D 总数 | E 中激活的 A-D | 比例 |
|---|---|---|---|
| **Layer 15** | 89 | 17 | **19.1%** |
| **Layer 24** | 85 | 12 | **14.1%** |
| **Layer 31** | 87 | 11 | **12.6%** |

A-D 特征中 **12-19% 在非代码过程拆解任务中同样激活**。这意味着 A-D 不是纯粹的"代码独有"，更准确的表述是"主要在代码中出现，但也部分覆盖通用过程拆解的特征"。

但这个标签修正对主结论没有影响：**无论叫"代码独有"还是"广义过程拆解"，negate 这些特征后推理不退化，这是不争的事实。**

### 关键补充：A-D 对代码生成也无因果作用

为进一步验证 A-D 特征的性质，我们测试了代码生成四个任务（Fibonacci、Quicksort、Palindrome、以及发现 A-D 时使用的原始 seed prompt）在 Layer 31 全量 negate 87 个 A-D 特征后的表现。结果：

| 代码生成任务 | Baseline | Negate A-D ALL 87 |
|---|---|---|
| Fibonacci 函数 | 正确实现 | **同样正确** |
| Quicksort 实现 | 正确实现 | **同样正确** |
| Palindrome 检查 | 正确实现 | **同样正确** |
| Seed prompt（定义 A-D 的种子 prompt） | 正确实现 | **同样正确** |

**连用来定义 A-D 的 seed prompt 的代码生成也不受影响。** 这意味着 A-D 特征对代码生成本身也无因果作用。这指向更深刻的结论：

```
A (代码生成 prompt 的 top-100 联合)
├── A-D = 代码"表面"特征（语法格式、编程习惯等）
│         → negate 对代码生成和推理都无影响
│
├── A∩D = 代码知识 ∩ 概念描述
│        ├── 计算概念知识（递归、排序、复杂度等）← 真正承载"能力"的部分
│        └── 通用语言特征（"是""有""一个"等）← 实验噪声
│        → negate 全面崩溃（通用语言特征主导，淹没了计算概念知识的信号）
```

**A∩D 是未解决的盲区：** 我们真正关心的"计算概念知识"可能藏在这里，但当前实验方法无法将其与通用语言特征分离。A-D 被排除，A∩D 被混淆。因此，"代码知识→推理迁移"这一假说在当前实验框架下**既不能证实，也不能彻底证伪**——它隐藏在 A∩D 中，等待新的实验方法。

### 最终判断

本实验通过三个逐步递进的阶段，全面检验了"代码拆解为推理提供 token 桥梁"假说：

| 阶段 | 关键操作 | 核心发现 | 结论 |
|---|---|---|---|
| **1. 相关性分析** | 扫描 code(30) vs description(15) prompt，建 A-D/A∩D | 代码独有特征(A-D)在推理中占比不突出；概念共享特征(A∩D)虽在 top-100 中占 15 个左右，但标准化后占比仍低 | 不支持假说，但无法排除因果 |
| **2. 排除概念共享控制** | 引入 D 描述控制组，验证 A∩D 是否为概念特征 | A∩D 让 D prompt 自身也大量激活（avg 41），但 A-D 在 D 上激活为 0 | A-D 分割正确，但 A∩D 疑似通用特征 |
| **3. 因果干预验证** | L31 A-D top-10; L31 A-D ALL 87; **跨层三层 A-D 261个**; L15 A-D ALL; 全部配 D-A 控制组 | **L31 A-D ALL 87: 无影响; 跨层 261: A-D 和 D-A 都崩; L15: A-D 和 D-A 都崩** | **代码特征无因果作用; 跨层/浅层干预崩溃是干扰剂量问题, 非代码特异性** |

**核心结论：**

```
1. A-D（代码专有特征）已被彻底排除。
   - 对推理无因果作用（Layer 31 全量 87 个 negate → 推理不受影响）
   - 对代码生成也无因果作用（seed prompt 的代码生成同样不受影响）
   - A-D 更可能是代码的表面语法特征（缩进、命名模式等），
     与代码的"计算能力"无关

2. A∩D 是未解决的盲区。
   - 包含我们真正关心的"计算概念知识"（递归、排序、复杂度等）
   - 但被通用语言特征淹没（negate → 全面崩溃，非特异性）
   - 当前 SAE top-100 方法无法分离两者

3. "代码知识→推理迁移"的假说：
   - 既不能证实（A-D 被排除，A∩D 被混淆）
   - 也不能彻底证伪（真正的计算概念特征可能藏在 A∩D 中）
   - 需要新的实验方法才能最终判定
```

**对原始假说的判决：在当前实验框架下未能找到支持证据。**

- **A-D（代码专有部分）被彻底排除**：不参与代码生成，也不参与推理
- **A∩D 是未解决的盲区**：计算概念知识若存在则藏在此处，但无法分离验证
- **SFT 增强推理应直接使用因果推理数据**，比通过代码增强更有效
- **方法学重要提醒**：SAE top-100 差异特征找到的是"最相关的"而非"最因果的"特征

### 方法学教训

| 问题 | 教训 |
|---|---|
| V1 用不相关领域做基线 | SAE 特征比较必须控制概念域 |
| top-100 的第 101 名问题 | 跨 prompt 扫频是必要的 |
| 候选集大小不同时比绝对数 | 必须标准化为比例 |
| 两个 prompt 集的交集 ≠ 概念共享特征 | **A∩D 可能是通用高频特征而非领域特定特征**。需控制组验证：如果 negate A∩D 导致完全无关的生成任务也崩溃，则说明这些特征是通用语言能力，而非领域概念 |
| seed-prompt 的差异 ≠ 全量扫描的差异 | `discovered_features.json` 基于两个 seed prompt 的差异，和 `rank_shared.py` 基于 30+15 prompt 全量扫描的 A∩D 结果有显著差异。**特征"跨界"是常见现象** |
| 跨层干预时，控制组无效 | Layer 15+24+31 同时 negate 261 个特征，**A-D 和 D-A 都崩**。不是代码特征特异性，而是总干扰量超过残差流容错阈值。跨层比较必须严格控制总干预数 |
| 不同层的容错能力不同 | Layer 31（最后一层）可承载 87 个 negate 无退化；Layer 15（第二层）89 个 negate 无论类型都崩。**干预验证应优先选择深层** |
| 相关性 ≠ 因果 | 需要干预验证 |

### 后续方向

当前实验在 Qwen3-8B 上得到了一致否定的结论，但以下方向仍有探索价值:

1. **其他模型族**: CodeLlama / DeepSeek-Coder 等代码训练比例更高的模型
2. **非 SAE 方法**: SAE top-100 的稀疏性可能错过分布在 #200-500 的关键特征
3. **激活放大**: 做 scale（放大代码特征）看推理是否改善，或可发现"有则更好，无则不变"的非对称关系
4. **A∩D 的进一步分析**: 当前 A∩D 被揭示为通用语言特征而非概念共享，可用海量文本的基准特征集先减掉通用特征再分析
5. **已完成的关键干预验证**（全部内容见完整证据链表）:
   - L31 A-D top-10 → 推理无影响 ✓
   - L31 A-D ALL 87（全量）→ **推理无影响 ★ 最重要证据** ✓
   - L31 A-D ALL 87 on **Hard Random Prompts**（排除背答案）→ **推理无影响 ✓**
   - L31 A∩D top-10 → 全面崩溃（通用语言特征）✓
   - L31 D-A top-3（控制）→ 无影响 ✓
   - L15 A-D ALL 89 → 崩溃；**D-A 控制组 89 也崩**（浅层脆弱性）✓
   - 三层 A-D ALL 261 → 崩溃；**D-A 控制组 261 也崩**（总剂量超阈值）✓
