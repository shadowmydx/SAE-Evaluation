# 实验报告：SAE 特征分离与代码→推理因果验证

**运行日期**: 2026-05-12
**模型**: Qwen3-8B + SAE-Res-Qwen3-8B-Base-W64K-L0_100
**实验框架**: code prompts (A, 30条) vs description prompts (D, 15条) 的 A∩D 共享特征分析

---

## 一、实验背景与目标

### 1.1 研究问题

大型语言模型的**代码生成能力**是否是其**文字推理能力**的因果来源？具体而言：SAE 特征空间中，在代码 prompt 中激活的特征，被抑制后是否会导致推理退化？

### 1.2 前期实验回顾（2026-05-11）

前一天完成了"推理重叠验证"实验，核心发现：

| 候选特征集 | 含义 | 结论 |
|-----------|------|------|
| A-D（代码独有特征） | 代码 prompt 顶集减去描述共享的"表面语法特征" | **Negate 全部 87 个不影响代码生成也不影响推理** |
| A∩D（共享特征） | 代码和描述都激活的"通用特征" | **Negate 全部 200 个导致全面崩溃**，但这是剂量效应+通用语言特征湮没，非特异性 |

关键遗留问题：A∩D 中的计算概念知识（如果存在）被通用语言特征淹没，无法分离。

### 1.3 本日实验设计（2026-05-12）

提出了一个改进方案：**不对 A∩D 全部取反，而是按"代码侧 vs 描述侧"的频率倾斜度（tilt）排序，挑选那些在代码 prompt 中明显更活跃（code-tilted）的特征**，然后：

1. 寻找正确的干预层
2. 使用更强信号的干预方式（add_direction）
3. 测试这些特征对代码生成和推理的因果作用

---

## 二、本日实验步骤与结果

### 2.1 Phase 1：发现干预层的错误

初始测试在 **Layer 31**（最后一层）进行。即使使用 30 个 code-tilted A∩D 特征、加 `add_direction α=-100`，代码生成仍完全正常。

通过 **Logits Lens** 发现了原因：

```text
Layer 19:  using (18%)            ← 代码结构开始形成
Layer 20:  using (80%)            ← 关键决策爆发点！
Layer 21:  using (94%)
Layer 23:  using (100%)
Layer 26:  fibonacci (80%)        ← 内容决策点
Layer 27-30: 逐步细化风格
Layer 31:  efficiently (45%)      ← 仅在决定措辞风格
```

**Layer 31 的干预发生在所有决策完成后**，干预无效的根本原因在此。

#### 残差流跳变幅度（L2 norm）

| 跳变区间 | 幅度 | 含义 |
|---------|------|------|
| L19→L20 | 34.17 | 结构决策（"using" 18%→80%） |
| L25→L26 | 86.27 | 内容决策（"fibonacci" 出现） |
| L28→L29 | 133.77 | 风格细化 |
| L33→L34 | 558.63 | LM head 前的最终调整 |
| L31 | — | **我们原来干预的位置** |

---

### 2.2 Phase 2：重写干预引擎

发现原有 `negate` 方式有根本性局限：

- 需要完整 SAE 编解码（残差→编码→修改 acts→解码→残差），有重构损失
- 信号幅度受限于原始激活值
- 取反后信号可能被重构误差淹没

新方法 `add_direction`：

```python
# 编码 → 确认 feature 当前是否在 top-100 中激活
# 如果激活：
#   1. 清空该 feature 在 SAE acts 中的贡献（避免双重计数）
#   2. 在残差空间直接加减 W_dec[:, fid] 方向向量
#     h' = h + α × W_dec[:, fid]
# 如果未激活：跳过（不做干预）
```

优势：无重构损失；α 可任意大；操作在残差空间（而非 SAE 空间）。

#### 控制实验

在 Layer 20 使用 **23 个随机 feature ID** 加 `α=-100` → **代码生成完全正常**。表明 add_direction 本身不产生噪声崩溃。

---

### 2.3 Phase 3：Layer 20 code-tilted A∩D 剂量滴定

**候选特征集**: Layer 20 A∩D 中 tilt ≥ 2.0 的 23 个特征。

| α | Fibonacci 代码 | 速度题 | 苹果题 | 序列题 | 结论 |
|---|-------------|--------|--------|--------|------|
| -5 | ✓ | ✓ | ✓ | ✓ | 安全阈值 |
| -10 | ✓ | ✓ | ✓ | ✓ | 安全阈值 |
| -15 | ✓ | **✗（句号重复）** | ✓ | **✗（？重复）** | **推理脆弱性出现** |
| -20 | ✓ | **✗（重复）** | ✓ | **✗** | 最佳选择性剂量 |
| -25 | **✗（ffff）** | 半崩 | ✓ | **✗** | 代码阈值到达 |
| -30 | **✗** | 部分恢复(?) | **✗** | 部分恢复(?) | 非单调区域 |
| -50 | **✗** | **✗** | **✗** | **✗** | 全面崩溃 |

**α=-20 是最佳选择性剂量**：在该剂量下代码生成基本正常，但复杂推理全面崩溃。

---

### 2.4 Phase 4：代码任务鲁棒性验证（α=-20）

| 代码任务 | Baseline | α=-20 | 结论 |
|---------|---------|-------|------|
| Fibonacci 函数 | 正常 | **正常** | 不受影响 |
| Palindrome 检查 | 正常 | **正常** | 不受影响 |
| Quicksort | 正常 | **0 输出（完全崩溃）** | **特异敏感** |
| Seed prompt | 正常 | **正常** | 不受影响 |

Quicksort 作为一个比 Fibonacci 更复杂的算法（需要 partition + 递归），对这类特征的抑制更敏感。表明特征的重要性随任务复杂度递增。

---

### 2.5 Phase 5：推理任务验证（α=-20）

| 推理任务 | Baseline | α=-20 | 结论 |
|---------|---------|-------|------|
| Stack LIFO | 正确 | **崩溃（重复）** | 因果作用确认 |
| Insertion Sort | 正确 | **崩溃（乱序重复）** | 因果作用确认 |
| Water Pouring | 正常推理 | **崩溃（卡在"4 is 4"循环）** | 因果作用确认 |
| 多步算术 | 正确 | **正常** | 例外—可能依赖其他路径 |

**4/5 推理任务被破坏**，且崩溃模式是重复/stuttering，不是随机噪声——这符合"推理链条断裂"的特征。

---

### 2.6 Phase 6：层特异性检验（α=-20）

相同 27 个 code-tilted 特征，但施加在 **Layer 26**（内容决策层）：

| 任务 | Layer 20 | Layer 26 |
|------|---------|---------|
| Fibonacci 代码 | 正常 | **正常** |
| Quicksort | **崩溃** | **正常** |
| 速度题 | **崩溃** | **正常** |
| 苹果题 | 正常 | **正常** |
| 序列题 | **崩溃** | **正常** |
| Stack LIFO | **崩溃** | **正常** |

**Layer 26 完全无效果。** 效应仅在 Layer 20（结构决策层）存在。说明这些 feature 代表的是**代码的"产出结构"特征**（如何在回答中编排推导步骤），而非代码的"内容知识"。

### 2.7 对照实验汇总

| 实验 | 层 | 特征 | 剂量 | 代码 | 推理 | 含义 |
|-----|----|------|------|------|------|------|
| 1 | 31 | Code-tilted A∩D ×30 | negate | 正常 | 正常 | 层不对，信号弱 |
| 2 | 31 | Code-tilted A∩D ×30 | add_dir α=-100 | 正常 | — | 层不对 |
| 3 | 20 | **随机** 23 features | add_dir α=-100 | **正常** | — | **不是噪声效应** |
| 4 | 20 | Balanced A∩D ×23 | add_dir α=-100 | 崩溃 | — | 通用高频特征→非特异性 |
| **5** | **20** | **Code-tilted A∩D ×23** | **add_dir α=-20** | **基本正常** | **全部崩溃** | **特异性因果作用** |
| 6 | 26 | Code-tilted A∩D ×27 | add_dir α=-20 | 正常 | 正常 | 仅 L20 有效 |

## 三、最终结论

**代码-倾斜的 A∩D 共享特征（code-tilted A∩D）在 Layer 20 上对推理存在特异性因果作用。**

### 3.1 已确认的事实

1. **存在性**：A∩D 中存在一批在代码 prompt 中显著更活跃的特征（tilt ≥ 2.0 的 23 个特征）
2. **层特异性**：这些特征仅在 Layer 20（逻辑输出的结构决策层）产生因果作用，不在 Layer 26 或 Layer 31
3. **因果方向**：抑制这些特征 → 推理崩溃（重复/stuttering/0 输出），且崩溃模式符合"推理链条断裂"
4. **选择性**：同等剂量下随机特征无影响 → 确认是特征特异性效应，而非噪声
5. **任务敏感性**：复杂推理（Stack、Insertion Sort、Water Pouring）比简单题（算术、苹果题）更脆弱
6. **代码影响**：简单代码（Fibonacci、Palindrome）不受影响，复杂代码（Quicksort）被完全抑制

### 3.2 假说地位

| 假说 | 状态 |
|------|------|
| "代码专有特征（A-D）是推理的来源" | **被否决**（negate 87 个无影响） |
| "A∩D 中的 code-tilted 特征参与推理" | **支持**（抑制后推理崩溃） |
| "这些特征是计算概念知识" | **需修正**：更像是"输出结构编排"特征，层特异性（仅 L20）支持这一解释 |

### 3.3 方法学要点

- **干预层选择必须基于 Logits Lens 分析**，默认选最后一个隐藏层是不正确的
- **negate 信号太弱**，add_direction（在残差空间直接加减 W_dec 方向向量）是更强的干预手段
- **剂量滴定 + 随机特征控制**是不可或缺的验证步骤
- 单一特征因果力太弱，需要按 tilt 分组联合干预

### 3.4 未解决的问题

- 多步算术题在 α=-20 仍正常：是否存在独立的算术计算通路？
- α=-30 的非单调区：是"第二路径救援"还是噪声？需要进一步验证
- 这些 L20 code-tilted 特征的具体语义是什么？（需要 SAE 词典查找 feature ID）

---

## Takeaway

**设定**：使用 Qwen3-8B + SAE，对 code prompts (30条) 和同概念 description prompts (15条) 进行 SAE top-100 特征扫描。取 A∩D 共享特征集（Layer 20，292个），按代码侧 vs 描述侧的激活频率比（tilt）排序，选出 top-23 code-tilted 特征。对其进行 `add_direction` 干预（在残差空间加减 W_dec 方向向量）抑制，观察代码生成和推理任务的退化。

**结论**：在 Layer 20 用 α=-20 抑制这 23 个 code-tilted A∩D 特征后，复杂推理（Stack、Insertion Sort、Water Pouring）全面崩溃，而同剂量随机特征无影响。简单代码（Fibonacci）不受影响，但复杂代码（Quicksort）被完全抑制。效应仅在 Layer 20（逻辑输出的结构决策层）存在，不在 Layer 26 或 31。这些特征更可能是"输出结构编排"特征而非纯"计算概念知识"特征，但 A∩D 中代码-倾斜特征对推理存在特异性因果作用已被确认。
